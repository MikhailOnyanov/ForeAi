Инструкция по работе

Уважаемый пользователь! Приложение Evallapp 1.0 предназначено для работы со статистическими данными для учёта и анализа дефектов в диаграммах классов с низким уровнем детализации языка UML (Unified Modeling Language).

В версии 2.0 будет реализован механизм создания пользовательских сущностей, что позволит загружать, анализировать и преобразовывать данные других предметных областей.

Данное руководство описывает основные элементы управления приложением и возможные ошибки.

Общие элементы страниц

Редактируемая таблица
Сформирована из данных первичной таблицы базы данных. При нажатии на ячейку можно отредактировать её содержимое – изменения будут зафиксированы в БД. Изменения применяются при потере фокуса клетки или при нажатии "Enter".

Колонки можно сортировать при нажатии на иконку заголовка.

Фильтрация по значениям происходит сразу после ввода текста в поле над таблицей.

Результат внесения изменений отображается в поле системных сообщений под таблицей. При нажатии на кнопку "Удалить" удаляется строка таблицы.

Возникновение ошибки:
- Изменить ячейку, которая является внешним ключом;
- Сделать пустой ячейку, которая не может быть пустой;
- Удалить ячейку, которая находится в отношении 1:N.
Список выбора таблиц
Составлен из т.н. первичных таблиц базы данных. При выборе элемента списка происходит обращение к подключённой БД, после чего выбранная таблицы отображается в приложении.
Импорт Excel файла
Реализует импорт содержимого файла в выбранную (из списка выбора таблиц) таблицу базы данных. В Evallapp 1.0 реализован только для зафиксированного формата файла.
Экспорт Excel файла
Выгружает все данные отображаемой таблицы в Excel файл в загрузки браузера.
